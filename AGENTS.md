## Build & Run

- **Framework**: SvelteKit + TypeScript
- **Styling**: Tailwind CSS v4 via `@tailwindcss/vite` (dark mode only)
- **Database**: SQLite via `better-sqlite3`
- **Search**: SQLite FTS5 (unicode61 tokenizer)
- **Package manager**: npm
- **Data ingestion**: `npm run ingest` (build-time JSON → SQLite into `data/archive.db`)
- **Dev server**: `npm run dev`
- **Build**: `npm run build` (must run `npx svelte-kit sync` first if `.svelte-kit/` missing)
- **Tests**: `npm test` (vitest)
- **Typecheck**: `npx svelte-check --tsconfig ./tsconfig.json`

## Project Structure


## Validation

- Tests: `npm test`

- Build: `npm run build`

## Operational Notes

- Source data lives in `temp-data/` (gitignored)
- SQLite DB at `data/archive.db`, generated by `npm run ingest`
- Single-user, read-only archive viewer
- Dark mode only — no light theme
- Tailwind v4 uses `@theme` block in `app.css` (not `tailwind.config.ts`)
- `vite.config.ts` imports from `vitest/config` (not `vite`) to support test config

### Codebase Patterns

- Color tokens in `src/app.css` `@theme` block (Claude dark theme)
- Date grouping uses Korean locale labels (오늘, 어제, etc.)
- FTS5 with unicode61 tokenizer for Korean text support
- UUID-based deduplication in data pipeline (INSERT OR REPLACE)
